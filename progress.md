# sa_plugin_db Progress

## Tracking Basis

- Scope: `db` plugin manifest, CLI command surface, qmod registry/execution, and local table storage helpers.
- Progress rule: update this file, including percentages, after each completed feature.
- Current status: `16 / 16` original qmod/CLI features completed (`100.0%`). Since that milestone, the plugin has moved into a development ABI phase with `db.sai` and `db.sal` exported through `sap.json` for SA-facing read handles, typed queries, row writes, dictionaries, blob stores, and ERP planner helpers. Row-list intersection, union, exclusion, candidate text/blob filtering, indexed two-predicate planners for `u64+i64`, `u64+u64`, `i64+i64`, `u64+blob_eq`, and `i64+blob_eq`, plus three-predicate planners for `u64+i64+blob_eq`, `u64+i64+bool`, `u64+i64+i64`, and semantic `u64+date+decimal_i64`, are now exposed for indexed AND/OR/not-in, text-search, and common ERP amount/date/status/business-key/flag composition. Strict fixed-width row update/upsert/delete by unique `u64`, `u8`, `i8`, `u16`, `i16`, `u32`, `i32`, and `i64` keys is available both directly and inside single-table transactions, and read handles can copy full rows by unique typed integer keys (`u64`, `i64`, `u32`, `i32`, `u8`, `i8`, `u16`, `i16`), unique `(u64, u64)` / `(u64, i64)` composite keys, and unique `blob_eq` text/blob keys for ERP form loads. Read handles now provide direct whole-column sums for both unsigned `u64` quantities and signed `i64`/scaled-decimal amounts, plus stable first-seen `u64` group counts and signed `i64` sums for full snapshots or filtered candidate row lists, including grouped `count/sum/min/max` pages and sorted pages by key/count/sum/min/max for Top N ERP reports. Unique `(u64, u64)` plus unique `(u64, i64)` composite-key upsert/update/delete is now available for ERP child rows and signed date/timestamp second-key shapes such as order lines, product/warehouse balances, customer/order-day lists, and status/due-day lists. Unique `blob_eq` text/blob business keys now support direct and transaction upsert/update/delete, with row `blob_handle` validation against the exact key bytes. Transaction-scoped dictionary intern is available through `sa_db_tx_dict_intern` / `DB_TX_DICT_INTERN`, allowing ERP status/category labels and the rows that reference them to commit or roll back together.
